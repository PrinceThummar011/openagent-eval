OpenAgent Eval●Overview Deck
Why this exists

Shipping AI without measuring it is flying blind

  • RAG quality depends on retrieval and generation — both fail silently.
  • Prompt tweaks move metrics you can't see.
  • Hallucinations and retrieval gaps slip into production.
  • Teams need reproducible, comparable experiments — not vibes.
  • Evaluation should be a test suite, not a spreadsheet.
The problem●OpenAgent Eval
What it is

One pipeline, every stage pluggable

A Config describes your dataset, retriever, LLM, and metrics. The Engine loads data, runs retrieval and generation, scores results, and persists a report.

Architecture

Provider-agnostic by design

  • Config — Pydantic models validate every run.
  • Engine + Pipeline — async retrieve → generate → score.
  • Executor — concurrent items with a worker pool.
  • Registry — resolves metrics & providers by name.

CLI

Six commands cover the whole loop

CLI●OpenAgent Eval
Python SDK

Embed evaluation in your tests

import asyncio from openagent_eval.config.models import Config from openagent_eval.core.engine import Engine config = Config(dataset={"path": "data/questions.json"}, llm={"provider": "openai", "model": "gpt-4o-mini"}) engine = Engine(config) report = asyncio.run(engine.run(dataset)) assert report.summary["failed_evaluations"] == 0

Quickstart

From install to insight in minutes

pip install openagent-eval oaeval init # write config.yaml oaeval run config.yaml # run evaluation oaeval report latest # inspect report

Offline? Use the built-in mock providers — no API keys required.
